1. Executive Summary

Regulatory Watch is organized as six sequential milestones over 22 weeks. This revised architecture (v2) preserves the spirit of the original plan while correcting three specific mistakes that would have caused significant over-engineering or cost blow-up at scale.

What changed from v1

#v1 positionv2 revisionRationale
1 Kafka + Zookeeper in infrastructure Redis broker only JVM + partition overhead adds nothing at MVP scale. Defer until a true multi-consumer event bus is actually needed.
2 RAG semantic search + vector embeddings included in M4 Removed from MVP RAG is a different product. Not required to detect or alert on regulatory change. Unblocks 1 week of M4 scope.
3 Single translation layer: translate all non-English docs → then NLP Split into input-side (OPUS-MT, Fast Track only) and output-side (DeepL/LLM, on summaries only, lazy & cached) LLMs read multilingual input natively — translating raw text before Deep Track is waste. Translating summaries only is ~150× cheaper than translating full documents.
4 Layer 3 (LLM Significance Score) shown but not specified Made explicit as keystone of M3 Gates alert volume. Without it, every whitespace change fires a ChangeEvent.
5 M4 is 6 weeks (Fast Track + Deep Track + RAG + embeddings) M4 is 4 weeks (Fast Track + Deep Track only) With RAG removed, scope fits comfortably in 4 weeks.

Cost impact of revisions

Estimated DeepL translation cost at 200 docs/day, 30% non-English: v1 ≈ $1,500/month (per-character translation of high-significance raw documents). v2 ≈ $10/month (translation of short summaries only, cached per (event, lang) pair). Savings: ~150×.

Translation is not a single layer. It is two different jobs, using two different tools, at two different points in the pipeline. Understanding this split is the most important architectural decision versus v1.

Deliberately NOT translated

  • Raw documents before Deep Track. Modern LLMs read multilingual input natively. Translating first introduces lossy paraphrase before summarization and costs ~150× more than the alternative.
  • Full raw text via DeepL. At $25 per million characters, a single 50-page rule translation costs $2–3. Reserved exclusively for the short output summary.

Milestone 5 — Translation & Personalization
Weeks 13–16 · 4 weeks effort · 0%

Goal. Multilingual support for international regulatory documents and a personalized alert engine that matches updates to individual user interest profiles.

Revision from v1 — the translation split

Translation is no longer a single layer that runs on all non-English raw documents. It is split into an input-side translator (OPUS-MT, Fast Track only) and an output-side translator (DeepL / LLM, on short summaries only, cached per event × language). The Deep Track LLM reads multilingual input natively and does not require translation.

User stories

  • US-5.1 Documents from non-English regulatory bodies usable by English-only NLP components.
  • US-5.2 Pharmaceutical analyst receives only pharmaceutical-related alerts.
  • US-5.3 Configure monitoring profile by industries, jurisdictions, regulatory bodies, keywords.
  • US-5.4 Alerts match conceptually-related regulations, not just exact keywords.
  • US-5.5 Choose notification frequency (immediate / daily digest / weekly digest).

12. Definition of Done

A milestone is considered done when all of the following are met:

  1. All tasks in the milestone have a merged pull request on main.
  2. All test scenarios for the milestone pass without manual workarounds.
  3. docker compose up produces a fully functional stack without errors.
  4. Code coverage for new modules is ≥ 70%.
  5. Documentation (README, docstrings, environment variables) is updated.
  6. No high-severity security vulnerabilities in dependencies (pip audit).

Closing note

This v2 revision preserves the disciplined, test-driven spirit of the original plan while eliminating two sources of over-engineering (Kafka, RAG-in-MVP) and correcting the single biggest cost risk (full-document translation). The resulting architecture is simpler, cheaper at scale, and remains professionally sound for the regulatory-watch problem domain.
