End-to-end workflow

From a source URL to a structured compliance alert — six stages.

01 · CRAWL

Fetch

Connector picks right handler. JS rendering for web, pdfplumber for PDFs.

02 · HASH

Persist

SHA-256 of body. Write raw_documents.

03 · DIFF

Version

If hash changed, new source_version and change_event.

04 · SCORE

LLM pass #1

Significance + topic + entities + origin country.

05 · EXTRACT

LLM pass #2

If score ≥ 0.5: actor/action/deadline/penalty rows.

06 · INDEX

Link

Entities, codes, country tags — ready for query & alert.

Key insight: every LLM call is gated. Unchanged content costs $0. Cosmetic changes stay cheap. We only pay for the signal.

LLM usage

Three narrowly-scoped jobs, each gated, each measured.

S

scoring

Runs on: every change event.
Returns: significance, topic, summary, entities, origin country, trade flow.
Cost: ~$0.0004 per call.

O

obligations

Runs on: score ≥ 0.5, substantive/critical only.
Returns: actor, action, deadline, penalty rows.
Cost: ~$0.00025 per call.

W

web_extract

Runs on: rare — noisy HTML only.
Returns: clean body text.
Cost: ~$0.0001 per call.

Gating makes the unit economics work: 53 scoring calls but only 27 obligation calls in the last run.

Evidence · 05

Country & trade-flow — who is affected?

  • origin_countries: LLM-extracted ISO-2 (CN, KR, ID…).
  • destination_countries: derived from source domain, always correct.
  • trade_flow_direction: inbound · outbound · bilateral · global.

Command

docker compose exec -T db psql -U regwatch -d regwatch -c "
SELECT substring(source_url FROM '[^/]+\.pdf\$') AS doc,
       significance_score, change_type, topic,
       origin_countries, destination_countries, trade_flow_direction
FROM change_events
WHERE origin_countries IS NOT NULL
ORDER BY significance_score DESC LIMIT 10;"
